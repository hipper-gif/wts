# Wts 一次記録インデックス（source manifest）

> **この文書の目的**: この案件の「一次記録はこれで全部」を1枚に列挙する。判断・回答・照合の前に、**ここに載っている正本を全部読んだか**を確認するためのチェックリスト。
> **なぜ要るか**: コンテキスト圧縮・要約で情報が落ち、「ここしか読んでませんでした」で誤判断する事故を防ぐ（lossy-context対策）。要約は正本ではない。判断前に必ず現物を読む。
> **網羅が要る問い**（「客に何を要望されたか」「何を約束したか」等）は、1文脈で全部要約せず、**正本1つにつき1リーダーで全読み→集約**（ファンアウト）する。

## 読み漏れ検知のルール

- 網羅が要る問いに答える前に、上表の該当行を**全部**開いたか確認する。飛ばした行があるならその旨を明記する（silent capを作らない）。
- 会話の要約・記憶で答えない。必ずこのインデックス経由で現物へ辿る。
- repo 外（別デバイス・Notion・Gmail・Google Chat等）にしか無い一次記録があるなら、ここに「repo外・要手動確認」と明記する。
